# Идеограф — Навигатор по идеям книг

Chrome-расширение для извлечения, структурирования и навигации по идеям из учебников и научных книг. Позволяет разбивать книгу на сеть связанных идей, отслеживать уровень освоения и быстро ориентироваться в материале.

## Проблема

Невозможно глубоко изучить все книги, которые хочется. Линейное чтение страница за страницей — медленно и неэффективно. Книги по одной теме (например, матанализу) на 80-90% пересекаются по содержанию. Ключевой навык — не прочитывать всё подряд, а **оперировать идеями**: понимать что уже знаешь, что новое, как идеи связаны между собой.

## Решение

Расширение, которое:
- Загружает PDF/DJVU книгу (через File System Access API, без хранения в IndexedDB)
- Анализирует выбранный диапазон страниц через LLM (текстовый или vision-режим)
- Извлекает идеи в структурированном виде (название, суть, тип, глубина, важность)
- Строит граф связей между идеями
- Позволяет отмечать статус каждой идеи (знакомо / не понятно / освоено)
- даёт быстрый обзор книги за 10-15 секунд по предисловию или оглавлению

## Ключевые принципы

1. **Идеи, не страницы** — книга рассматривается как набор идей со связями, а не линейный текст
2. **VLM-экстракция** — книги без текстового слоя (сканы, DJVU) обрабатываются через vision-модели
3. **Нет встроенного ридера** — чтение во внешнем приложении (zathura и др.), расширение — «мозговой центр»
4. **Результат на русском** — извлечение идей из книги на любом языке, результат всегда на русском
5. **Селективный анализ** — обрабатываются только выбранные пользователем участки, не вся книга целиком

### Модули

#### 1. Библиотека книг
- Список загруженных книг с metadata (название, автор, кол-во страниц, формат)
- Подключение книги через File System Access API (`showOpenFilePicker`)
- Хранение `FileSystemFileHandle` в IndexedDB (весит ~100 байт, не файл)
- Автоопределение формата (PDF / DJVU)
- Если permission истёк — кнопка «Переподключить»
- Интеграция с внешним ридером: копирование команды `zathura -f N /path/to/file`

#### 2. Экстракция идей
- Выбор диапазона страниц для анализа
- Автоопределение режима: текстовый слой или VLM (vision)
  - Пробуем извлечь текст с первой страницы
  - Если текст < 50 символов или нечитаемый → VLM-режим
  - Пользователь может переключить вручную
- **Текстовый режим**: PDF.js → текст → отправка в LLM
- **VLM-режим**: рендер страницы в canvas → PNG → base64 → vision LLM
- Двухпроходная обработка:
  - Проход 1: извлечение идей из каждого чанка
  - Проход 2: построение связей между всеми идеями диапазона
- Результат всегда на русском языке (указывается в промпте)

#### 3. Карточки идей
- Список идей с фильтрами (по статусу, глубине, типу, книге)
- Карточка каждой идеи: название, суть, тип, важность, страница
- Интерактивные статусы: Не изучал → В процессе → Освоено → Применяю
- Знакомство: Знаю глубоко / Слышал / Не знаком
- Пользовательские заметки и вопросы к каждой идее
- Переход к странице книги: копирование команды для внешнего ридера

#### 4. Граф идей
- Force-directed граф на vis-network
- Визуальные кодировки:
  - **Форма узла**: ● определение, ■ метод, ◆ теорема, ▲ инсайт
  - **Цвет узла**: 🟢 освоено, 🟡 в процессе, 🔵 знакомо, ⚪ не изучал, 🔴 не понятно
  - **Размер**: важность × (3 - уровень знакомства)
  - **Толщина ребра**: 1px аналог, 2px развивает/применяет, 3px prerequisite
- Клик по узлу → карточка идеи
- Drag-and-drop, zoom/pan
- Фильтры и поиск

#### 5. Настройки (модальное окно)
- API ключи: OpenRouter, z-ai (расширяемо на GigaChat, Kimi, MiniMax)
- Выбор активного провайдера и модели
- Тема: светлая / тёмная / системная
- Язык интерфейса
- Детализация извлечения: низкая / средняя / высокая
- Экспорт/импорт базы идей

### Модель данных

```typescript
// ============ Книга ============
interface Book {
  id: string;                    // UUID
  title: string;                 // из metadata или имя файла
  author: string;                // из metadata
  totalPages: number;            // общее число страниц
  format: 'pdf' | 'djvu';       // автоопределённый формат
  extractionMode: 'text' | 'vlm'; // авто или вручную
  filePath?: string;             // путь для интеграции с ридером
  fileHandle?: FileSystemFileHandle; // для File System Access API
  tableOfContents: TOCEntry[];   // оглавление (если доступно)
  lastAnalyzedPage: number;      // до какой страницы проанализировано
  createdAt: number;
  updatedAt: number;
}

interface TOCEntry {
  title: string;
  page: number;
  level: number;                 // 1 = глава, 2 = секция, 3 = подсекция
  parentId?: string;
}

// ============ Идея ============
type IdeaType = 'definition' | 'method' | 'theorem' | 'insight' | 'example' | 'analogy';
type IdeaDepth = 'basic' | 'medium' | 'advanced';
type IdeaImportance = 1 | 2 | 3 | 4 | 5;
type Familiarity = 'unknown' | 'known' | 'heard' | 'new';
type IdeaStatus = 'unseen' | 'in_progress' | 'mastered' | 'applied' | 'confused';
type RelationType = 'prerequisite' | 'elaborates' | 'contradicts' | 'analogous' | 'applies';

interface Idea {
  id: string;                    // "bookId_page_idx" или UUID
  bookId: string;
  chapterId?: string;            // привязка к главе

  // Содержание
  title: string;                 // краткое название (до 10 слов)
  summary: string;               // суть в 1-2 предложениях
  quote?: string;                // ключевая цитата (если есть текстовый слой)

  // Классификация
  type: IdeaType;
  depth: IdeaDepth;
  importance: IdeaImportance;

  // Позиция в книге
  pages: number[];               // на каких страницах упоминается

  // Состояние пользователя
  familiarity: Familiarity;
  status: IdeaStatus;

  // Пользовательский контент
  notes: string;
  questions: string[];
  userTags: string[];

  // AI-мета
  aiModel: string;               // какая модель извлекла
  provider: string;              // OpenRouter, z-ai и т.д.
  extractedAt: number;

  // Связи (denormalized для быстрого доступа)
  relations: Relation[];
}

interface Relation {
  targetId: string;
  type: RelationType;
  description?: string;          // краткое описание связи
}

// ============ Настройки ============
interface Settings {
  activeProvider: 'openrouter' | 'z-ai';  // расширяемо
  providerKeys: {
    openrouter?: string;
    'z-ai'?: string;
    gigachat?: string;
    kimi?: string;
    minimax?: string;
  };
  activeModel: string;           // slug модели на активном провайдере
  theme: 'light' | 'dark' | 'system';
  language: 'ru';
  extractionDetail: 'low' | 'medium' | 'high';
}

// ============ Кэш текста ============
interface PageTextCache {
  bookId: string;
  pageNumber: number;
  text: string;                  // извлечённый текст (если есть)
  hasTextLayer: boolean;         // есть ли текстовый слой
  imageBase64?: string;          // рендер страницы (для VLM-режима)
  cachedAt: number;
}
```

## Пайплайн обработки

```
 1. Пользователь подключает книгу (File System Access API)
      │
      ▼
 2. Определение формата (PDF/DJVU) и количества страниц
      │
      ▼
 3. Попытка извлечь текст с первой страницы
      │
      ├─ Текст есть (> 50 символов, читаемый) ──→ Текстовый режим
      │
      └─ Текст отсутствует или нечитаемый ──→ VLM-режим
      │
      ▼
 4. [Опционально] Быстрый скан предисловия / оглавления
      │  → LLM-анализ → список ключевых идей книги
      │  → Пользователь решает: стоит ли углубляться
      ▼
 5. Пользователь выбирает диапазон страниц (from – to)
      │
      ▼
 6. Извлечение контента для каждой страницы
      │
      ├─ Текстовый режим:
      │    PDF.js textLayer → текст страницы
      │
      └─ VLM-режим:
           Рендер в скрытый canvas → PNG → base64
      │
      ▼
 7. Группировка в чанки (~3000-5000 токенов для текстового,
    по 1-2 страницы для VLM)
      │
      ▼
 8. Проход 1: Извлечение идей (каждый чанк → LLM)
      │
      │  Prompt: извлечь идеи, вернуть JSON
      │  Результат на русском
      │
      ▼
 9. Дедупликация и мерж идей
      │
      ▼
10. Проход 2: Построение связей (все идеи диапазона → LLM)
      │
      │  Prompt: определить связи между идеями
      │
      ▼
11. Сохранение в IndexedDB
      │
      ▼
12. Отображение в списке идей / графе
```

## Стек технологий

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| Расширение | Chrome Manifest V3 | Современный стандарт, стабильный API |
| PDF-рендер | PDF.js (Mozilla) | Факт-стандарт, текстовый слой + canvas |
| DJVU-рендер | djvu.js | JS-библиотека для рендера DJVU в canvas; fallback — Linux-утилиты (djvulibre) |
| Граф идей | vis-network (vis.js) | Force-directed граф с physics engine из коробки, меньше кода чем D3 |
| UI | Vanilla JS + CSS Variables | Для расширения не нужен тяжёлый фреймворк |
| Хранение | IndexedDB через Dexie.js | Типобезопасный доступ, промисы, простой API |
| AI | OpenRouter API (fetch из background) | Доступ к Claude, GPT-4o, Gemini, Llama и др. |
| Сборка | Vite + CRXJS | Быстрая hot-reload разработка для расширений |
| Тема | CSS Variables + `prefers-color-scheme` | Тёмная/светлая с переключателем |

## Спецификация LLM-промптов

> Подробные промпты будут в `docs/prompts.md` после фазы 0. Ниже — общая стратегия.

### Проход 1: Извлечение идей (текстовый режим)

```
Системный промпт:
  Ты — аналитик знаний. Извлекай из текста самостоятельные идеи,
  концепции, определения, утверждения, методы.
  Пиши результат на русском языке.

  Для каждой идеи укажи:
  - title: краткое название (до 10 слов)
  - summary: суть в 1-2 предложениях
  - type: definition | method | theorem | insight | example | analogy
  - depth: basic | medium | advanced
  - importance: 1-5 (насколько фундаментальна)
  - pages: номера страниц (из контекста)
  - quote: ключевая цитата (если есть)

  Правила:
  - НЕ дроби мелкие детали
  - НЕ дублируй повторяющиеся идеи
  - Если идея опирается на другую — укажи в requires[]

  Верни JSON массив идей.
```

### Проход 1: Извлечение идей (VLM-режим)

```
Системный промпт:
  Ты — аналитик знаний. На изображении — страница учебника/книги.
  Извлеки все идеи с этой страницы.
  Пиши результат на русском языке.
  Формат тот же, что и для текстового режима.
```

### Проход 2: Построение связей

```
Системный промпт:
  Дан список идей из книги. Определи связи между ними:

  Типы связей:
  - prerequisite: идея A нужна для понимания B
  - elaborates: идея A развивает/уточняет B
  - contradicts: идея A противоречит B
  - analogous: идея A аналогична B из другой области
  - applies: идея A применяется в B

  Верни JSON массив: [{source, target, type, description}]
```

## Решения по ключевым вопросам

### Хранение файлов: File System Access API

**Почему не IndexedDB для файлов:**
- Лимит IndexedDB: 50-500MB (зависит от браузера и диска)
- Типичный учебник: 10-100MB, DJVU: 5-50MB
- 10-20 книг могут заполнить лимит

**Почему не Native Messaging:**
- Требует установки отдельной программы на компьютер
- Разная установка под Windows/Linux/macOS
- Избыточно для нашей задачи

**Решение:** `FileSystemFileHandle` хранится в IndexedDB (~100 байт). При необходимости чтения — `handle.getFile()` → ArrayBuffer → рендер. Книга остаётся на диске,IndexedDB не перегружается.

** permission истекает** — обычно после закрытия браузера. При повторном открытии расширения — кнопка «Переподключить файл», которая запрашивает новый handle и обновляет запись в БД. Путь файла также хранится для справки (но через `resolve()` нельзя восстановить handle по пути).

### DJVU-рендер: стратегия

1. **Основной путь**: djvu.js (JavaScript-библиотека для рендера в canvas)
2. **Fallback**: Если djvu.js не справляется с конкретным файлом — предложить пользователю конвертацию через Linux-утилиты:
   - `djvups book.djvu book.ps && ps2pdf book.ps book.pdf`
   - `ddjvu -format=pdf book.djvu book.pdf`
3. **OCR fallback**: Если ни текстовый слой, ни VLM не дают результат — предложить `tesseract` через Native Messaging (далёкая перспектива)

### Интеграция с внешним ридером

Кнопка «Открыть в ридере» → копирует в буфер обмена команду:

```bash
zathura -f 42 /home/user/books/math_analysis.djvu
```

Пользователь вставляет в терминал одним кликом. Не требует установки native host, работает на любом Linux.

В перспективе: можно зарегистрировать `ideograph://` protocol handler через `registerProtocolHandler()`, но это требует desktop-файла и более сложной настройки.

### Мультиязычность книг

Язык книги не важен. В промпте указано: «пиши результат на русском языке». LLM (особенно GPT-4o, Claude) отлично извлекает идеи с английского, немецкого, французского и других языков и формулирует на русском. Не нужно делать перевод книги — только извлечение идей на русском.

### Автоопределение режима (text vs VLM)

```javascript
async function detectExtractionMode(book, pageNumber) {
  const text = await extractTextFromPage(book, pageNumber);
  const cleaned = text.replace(/\s/g, '').length;
  const readableChars = text.match(/[\p{L}\p{N}]/gu)?.length || 0;
  const ratio = readableChars / Math.max(cleaned, 1);

  // Если мало читаемых символов — скорее всего мусорный OCR или его нет
  if (cleaned < 50 || ratio < 0.3) {
    return 'vlm';
  }
  return 'text';
}
```
